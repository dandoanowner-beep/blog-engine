# ADR-006: Denormalized feed_score for Explore Feed Performance
**Date:** 2026-05-30 | **Status:** Accepted

## Problem
The Explore feed ranking algorithm requires sorting by a combination of recency, likes, comments, and follow-boost. Computing this on every read would require expensive joins across multiple tables.

## Decision
Store a `feed_score` column directly on the `blogs` table. Recalculate and update it on every like, dislike, or comment event. Feed query simply does `ORDER BY feed_score DESC`.

## Score Formula
```
feed_score = (like_count * 3) + (comment_count * 2) + recency_bonus
recency_bonus = max(0, 100 - hours_since_published * 2)
```
Follow-boost (50 points) is applied at query time via a LEFT JOIN on follows table (not stored).

## Consequences
- Feed reads are O(1) on the score column (indexed)
- Score updates add ~1ms overhead on each like/comment write — acceptable
- Score naturally decays as blogs age (recency_bonus reduces over time)
- Denormalization means score can drift slightly under concurrent writes — acceptable for feed ranking (not financial data)
