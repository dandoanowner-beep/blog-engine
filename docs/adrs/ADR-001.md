# ADR-001: Go as Backend Language
**Date:** 2026-05-30 | **Status:** Accepted

## Problem
Choose a backend language for the blog engine API.

## Decision
Use Go (Golang).

## Reasons
- Owner has Go expertise (existing knowledge)
- Excellent performance for concurrent HTTP handling
- Strong standard library (net/http, crypto, encoding)
- Statically typed — catches bugs at compile time
- Single compiled binary — simple deployment

## Consequences
- Frontend and mobile cannot share backend code (expected)
- Team must be Go-proficient (satisfied — owner knows Go)
