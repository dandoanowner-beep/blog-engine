# ADR-009: Frontend i18n — react-i18next with JSON Locale Files
**Date:** 2026-06-07
**Status:** Accepted
**Sprint:** i18n-bilingual (delta)

---

## Problem

All frontend UI text is currently hardcoded in English. The blog targets Vietnamese readers (default) and English readers. We need a maintainable way to switch all static UI text between Vietnamese and English without page reload.

---

## Decision

Use `react-i18next` (the React binding for `i18next`) with static JSON locale files (`vi.json`, `en.json`). Language preference is stored in `localStorage` via i18next's built-in language detector. Default language is Vietnamese. The `LanguageToggle` component in the header calls `i18n.changeLanguage()`, which re-renders all translated strings immediately.

---

## Alternatives Considered

| Option | Pros | Cons | Decision |
|--------|------|------|----------|
| Manual string constants (own solution) | Zero dependencies | No hot-switch; requires re-render plumbing | Rejected |
| `next-intl` | Great DX | Next.js only; project uses Vite | Rejected |
| `formatjs` / `react-intl` | ICU message format | More complex setup; overkill for 2 languages | Rejected |
| **react-i18next (chosen)** | Industry standard; hot-switch; localStorage persistence; tree-shakeable | Adds ~35KB to bundle | **Accepted** |

---

## Consequences

**Positive:**
- `t('key')` pattern is familiar to all React developers
- Language switches instantly without page reload or API call
- `localStorage` persistence means preference survives browser close
- JSON locale files are easy to update or hand off to a translator

**Negative / Trade-offs:**
- ~35KB bundle size increase (acceptable; gzip reduces to ~12KB)
- All hardcoded strings in existing pages/components must be replaced with `t('key')` — large but mechanical refactor
- Missing translation key falls back to key string (not ideal visually); mitigated by testing all keys exist in both locale files

**Implementation Notes:**
- `i18n.ts` initializes i18next with `initReactI18next`, loads both locale files synchronously (small files, no lazy load needed)
- Language detector order: `localStorage` → fallback `'vi'`
- Locale key naming convention: `namespace.component.element` (e.g. `nav.explore`, `blog.publish`, `auth.signIn`)
