# ADR-008: Translation Service — Claude API with Async Goroutine
**Date:** 2026-06-07
**Status:** Accepted
**Sprint:** i18n-bilingual (delta)

---

## Problem

Blog content is authored in Vietnamese. English-speaking readers need to read the same content in English. We need a reliable, cost-effective translation mechanism that does not degrade the blog author's publish experience or block the API response.

---

## Decision

Use Claude API (`claude-sonnet-4-6`) to translate blog title and body from Vietnamese to English. Translation is triggered asynchronously in a goroutine after the blog is saved to the database. The API response (HTTP 201/200) is returned to the client immediately — translation happens in the background. Translated content is stored in `title_en` and `body_en` columns. A `translation_status` column tracks state: `none` → `pending` → `done` / `failed`.

---

## Alternatives Considered

| Option | Pros | Cons | Decision |
|--------|------|------|----------|
| Google Translate API | Cheap, fast | Lower quality for VI/EN nuance; paid after 500k chars/month | Rejected |
| DeepL API | Good quality | No Vietnamese language support | Rejected |
| Claude API — synchronous (block response) | Simple code | Adds 2–5s latency to every blog publish; poor UX | Rejected |
| Author writes both manually | Perfect quality | Author must write every post twice; high friction | Rejected |
| **Claude API — async goroutine (chosen)** | No publish latency; high quality VI/EN; stored result = zero per-read cost | Goroutine errors need careful logging; slight eventual consistency | **Accepted** |

---

## Consequences

**Positive:**
- Blog publish is instant — no waiting for translation
- Translation stored in DB — zero cost per reader request after first translation
- `claude-sonnet-4-6` produces high-quality Vietnamese→English output
- Graceful degradation: `translation_status=failed` shows VI content with notice instead of error

**Negative / Trade-offs:**
- Brief window where newly-published blog shows VI content even if language=EN (translation in-flight)
- Goroutine panics must be recovered (`defer recover()`) to avoid crashing the server
- New `ANTHROPIC_API_KEY` env var required in deployment

**Constraints:**
- Goroutine uses a background context (not request context) to survive after HTTP response is sent
- Translation failure must never prevent the blog from being saved
- Re-translation triggered on edit only if `title` or `content` fields actually changed
