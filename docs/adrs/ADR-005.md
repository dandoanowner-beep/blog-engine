# ADR-005: Server-Side HTML Sanitization (XSS Prevention)
**Date:** 2026-05-30 | **Status:** Accepted

## Problem
Blog content is user-submitted HTML from the WYSIWYG editor. Rendering it unsanitized creates XSS vulnerabilities (Risk R-01 in RISK_REGISTER.md).

## Decision
Sanitize all blog HTML server-side using the `microcosm-cc/bluemonday` Go library before storing in the database. Never trust client-submitted HTML.

## Allowed HTML elements
- Text: `<p>`, `<br>`, `<strong>`, `<em>`, `<u>`, `<s>`
- Headings: `<h1>` through `<h4>`
- Lists: `<ul>`, `<ol>`, `<li>`
- Code: `<code>`, `<pre>`
- Blockquote: `<blockquote>`
- Images: `<img src>` (src must be relative path to uploaded image — no external URLs)
- Links: `<a href>` (href must be http/https only, target="_blank" allowed)
- Divider: `<hr>`

## Stripped elements
All other tags and attributes, especially: `<script>`, `<style>`, `<iframe>`, `on*` event attributes, `javascript:` URLs.

## Consequences
- Sanitization adds ~1ms per save (negligible)
- Some advanced formatting edge cases may be stripped — acceptable trade-off for security
