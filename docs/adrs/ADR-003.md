# ADR-003: JWT Authentication with httpOnly Cookies
**Date:** 2026-05-30 | **Status:** Accepted

## Problem
Choose an authentication mechanism that is both secure and works across web and mobile clients.

## Decision
JWT access tokens (15min TTL) + refresh tokens (7 days), stored in httpOnly cookies on web. Mobile stores tokens in secure storage (React Native Keychain).

## Reasons
- Stateless API — no server-side session store required
- httpOnly cookies prevent XSS token theft on web
- Short access token TTL limits damage from token leak
- Refresh token rotation keeps sessions alive without re-login

## Consequences
- Mobile must use device secure storage (not cookies)
- API must handle CORS properly for cookie-based auth
- Token revocation requires a blocklist (add in Sprint 2 if needed)
